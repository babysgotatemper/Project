GIT

Розподілена система керування версіями файлів та спільної роботи

Проект створив Лінус Торвальдс для управління розробкою ядра Linux, а сьогодні підтримується Джуніо Хамано (англ. Junio C. Hamano). Git є однією з найефективніших, надійних і високопродуктивних систем керування версіями, що надає гнучкі засоби нелінійної розробки, що базуються на відгалуженні і злитті гілок. Для забезпечення цілісності історії та стійкості до змін заднім числом використовуються криптографічні методи, також можлива прив'язка цифрових підписів розробників до тегів і комітів.

Налаштування локального репозиторію

  • https://githowto.com/ інформація по системі
  • $ git config --global --list потрібно налаштувати username i email ($ git config --global user.name `ім’я`, $git config --global user.email "your_email@whatever.com")
  • створити каталог нового проекту ($ mk dir папка), або зайти внього ($ cd папка/папка)
  • проініціалізувати файли проекту через $ git init, для перевірки використати $ git status
  • для того, щоб GIT ігнорував певні папки або файли існує або потрібно створити файл (.gitignore) в якому вказати шлях до відповідного файлу/каталогу, для коментарів використовується символ (#)
  • після внесення змін у файлову структуру проекту їх необхідно внести до локального репозиторію, щоб в подальшому система за ними слідкувала ($ git add . або $git add назва_файлу.розширення)
  • для внесення коментарів про проведену роботу і над проектом існує команда $ git commit -m “опис”. Для перегляду історії коммітів $ git log

Робота з GITHUB

  • Для початку потрібно зараєструватись на https://github.com/
  • Потім створити новий репозиторій для проекту (Create a new repository +)
    • вказати назву і опис проекту(необов’язково)
    • на Public (Private платна опція)
    • Initialize this repository with a README (те саме що і git init в консолі)
  • Далі з’явиться 2 варіанти підключання локально репозиторія до github
    • створити новий +команди
    • підключити вже існуючий +команди
  • Коли все підключено, залишилось лиш вносити зміни в проект і заливати їх на github за допомогою $ git push
  • Якщо потрібно стягнути з github вже внесені кимось правки або весь проект у свій репозиторій $ git pull
