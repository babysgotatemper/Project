Теги для відправки форм (для вводу)

Для передачі користувацьких даних на сервер в HTML існують форми. З їх допомогою реалізовують все: від листування користувачів до форумів і повноцінних веб-додатків. Форма складається з елемента , усередині якого розташовується будь-який вміст (крім інших форм), що включає поля введення, кнопки, текстові поля, випадаючі списки і т.д.

Input tag

Найбільш використовуваним елементом форм є

З його допомогою створюються поля для введення тексту, паролів і вибору файлів, а також кнопки, прапорці та перемикачі.

У HTML 5 він ще і пристосований для введення всіляких дат, числових значень, телефонів, адрес і навіть вибору кольору. Все це різноманіття визначається атрибутом type

Input tag

  • Атрибут type встановлює тип тегу

  • "text" - значення по замовчуванню. Елемент призначений для введення текстового рядка.

  • "button" - кнопка, дія якої не визначено, value це назва кнопки.

  • "reset" - кнопка очищення форми.

  • "submit" - кнопка відправки даних на сервер.

  • "checkbox" - «вкл / викл». елемент містить логічний атрибут checked = "checked".

  •  

    "radio" - перемикач. В формі повинно бути кілька елементів з однаковим атрибутом name="radioGroup_name" , при кліку активовуватись буде тільки один.

  • Для організації випадаючих списків використовують структуру, що складається з елемента "select" , усередині якого розміщуються дочірні "option" , що представляють варіанти вибору.
  • Передане на сервер ім'я поля вказується в атрибуті name елемента "select" , а його значення - в ​​атрибуті value елемента "option" .
  • Щоб вказати варіант за замовчуванням застосовується інший логічний атрибут - selected="selected" .

Label tag


  • Ще одним елементом, який прив'язується до інших, є "label". Його значення не відправляється на сервер.
  • В атрибуті for вказується ім’я поля введення для прив’язки.
  • Якщо "label" пов'язаний з полем input type = "checkbox", то прапорець буде змінювати своє значення не тільки при натисканні по ньому, але і з цього "label" .

Tе ж саме буде якщо не вказувати атрибут for і помістити тег input у відповідне йому поле label
